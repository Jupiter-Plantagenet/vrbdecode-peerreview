# eval/artifacts

This folder contains **generated, persisted artifacts** for VRBDecode evaluation runs.

Layout (per run)
- `eval/artifacts/<run_id>/meta.json`
- `eval/artifacts/<run_id>/protocol_version.txt`
- `eval/artifacts/<run_id>/vectors_hash.txt`
- `eval/artifacts/<run_id>/public_inputs.json`
- `eval/artifacts/<run_id>/receipt.json`
- `eval/artifacts/<run_id>/nova/ivc_proof.bin` (compressed `IVCProof`; enables wrap without re-folding)
- `eval/artifacts/<run_id>/wrapped/verifier.sol`
- `eval/artifacts/<run_id>/wrapped/calldata.bin`
- `eval/artifacts/<run_id>/wrapped/proof.bin` (compressed decider proof; used to regenerate calldata/debug)
- `eval/artifacts/<run_id>/wrapped/wrapped_metrics.json`
- `eval/artifacts/<run_id>/chain/*` (local chain deploy/verify receipts; may be empty until enabled)
- `eval/artifacts/<run_id>/baselines/*`
  - `baseline_groth16_batch_fcircuit_k*_b*_verifier.sol`
  - `baseline_groth16_batch_fcircuit_k*_b*_calldata.bin`

Notes
- `run_id` is generated by the pipeline; do not hand-edit run directories.
- All runs should be reproducible from scripts/binaries in this repo. `meta.json` is expected to include the code revision and environment versions used.
- Expensive preprocessing is cached under `eval/cache/` by default (can be changed with `pipeline --cache-dir ...`); it is safe to delete, but will slow subsequent runs.

Convenience runner
- `python3 eval/run_submission.py --run-id <run_id> --k 16 --n 2 --verify-anvil`
  - Writes `eval/artifacts/<run_id>/summary.json` aggregating pipeline + (optional) Anvil + baseline outputs.
